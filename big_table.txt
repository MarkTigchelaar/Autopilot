Tables:



ObjectTable
 - id (index)
 - object_ref


// starting point for type checking.
Typename:
 - name (like MyStruct etc)
 - category (any kind of user defined type, especially one that defined in define statement) [module, import, struct, enum, error, union, define]
 - module_id



Module table:
 - module,
 - path(no filename, since modules occupy one directory level)
 - id

File table:
 - filename
 - module_id

ImportTable
 - module_id
 - filename
 - object_id
 - import_type
 - path_list
 - import_items


Enumerable table:
// any list like thing, like enums, errors, unions, structs, function headers, interfaces
 - id
 - name <- typename table has this
 - typename
 - list_items (list of type Enumerable)
 - method_list (list of methods, or headers for interface) or None


Enumerable type:
 - name
 - type name
 - public
 - let_or_var (or None)
 - field or arg or return type or interface (use strings for these values)
 - default_value


Modifier table:
 - item_id
 - modifier list [pub, inline, acyclic]
